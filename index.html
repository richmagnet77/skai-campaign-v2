Phase 0의 의의 — 응급조치

두 사이트 모두 JavaScript SPA(Single Page Application) 구조로, 네이버봇·GPTBot·ClaudeBot 등 AI 크롤러가 본문을 읽지 못하는 상태입니다. 근본 해결(SSR/SSG 전환, Phase 2)은 2~6주가 필요하지만, Phase 0의 4가지 작업은 합쳐서 2~3일이면 즉시 효과가 납니다.

네이버봇 — JS 미실행, 본문 색인 불가 GPTBot·ClaudeBot·PerplexityBot — AI 검색 인용 누락 구글봇 — 렌더링 지연, sitemap 없어 색인 효율 저하

0-1
SPA 폴백 정책 긴급 수정
담당: AX랩
공수: 0.5일 · 난이도: 낮음
문제: 서버가 /sitemap.xml 포함 모든 경로에 동일한 빈 HTML을 반환 → 검색엔진이 "사이트맵 깨짐"으로 판단, soft 404 대량 발생

목표

  • 정적 파일(/sitemap.xml, /robots.txt 등)은 실제 파일 반환 또는 404 응답
  • 존재하지 않는 페이지는 반드시 HTTP 404 상태 코드로 응답

환경별 적용 방법

환경방법
Next.js + Vercelvercel.json rewrites에 정적 파일 경로 negative lookahead 추가
Next.js + Nginxlocation 블록에서 /robots.txt, /sitemap.xml 먼저 try_files로 분리
AWS S3 + CloudFrontCloudFront Function으로 정적 파일 경로 폴백 제외 처리
Apache (.htaccess)RewriteCond로 정적 파일 경로 우선 매칭 후 그 외만 index.html 폴백

완료 기준

  • /sitemap.xml → HTTP 200 + application/xml
  • /robots.txt → HTTP 200 + text/plain
  • /존재하지않는경로 → HTTP 404
  • / (홈) → HTTP 200 정상 유지
